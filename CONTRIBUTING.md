# 多人协作说明

这个项目推荐使用 Git 和 GitHub 进行协作。

## 建议流程

1. 每个人先从主分支拉取最新代码
2. 新功能在自己的分支开发
3. 开发完成后提交到远程仓库
4. 发起 Pull Request
5. 由其他协作者检查后再合并

## 分支命名建议

- `feature/功能名`
- `fix/问题名`
- `docs/文档名`

## 提交信息建议

- `feat: 添加登录页`
- `fix: 修复首页显示问题`
- `docs: 更新项目说明`

## 注意事项

- 不要直接在 `main` 分支上长期开发
- 提交前先同步最新代码
- 不要把密码、密钥等敏感信息提交到仓库
